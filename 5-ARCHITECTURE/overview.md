# Architecture Overview вҖ” LEO Fashions

## System Architecture

LEO Fashions is a **multi-tier web and mobile application** hosted on the cloud. It follows a standard client-server architecture with role-based access control.

## Key Architectural Decisions

**1. Role-Based Access Control (RBAC)** Every API endpoint checks the user's role before processing. Admin gets full access. User gets filtered access to their own data only.

**2. Approval Workflow** A dedicated approval queue separates creation from activation. This prevents any unauthorized action from taking effect.

**3. Auto ID Generation** All record IDs are auto-generated by the server, ensuring uniqueness and consistent formatting across all modules.

**4. Soft Relationships** All records are linked by ID references (order_id, supplier_id etc.), making data retrieval flexible and auditable.

**5. Immutable Audit Trail** Every data change is logged with who made it and when, supporting full traceability.
